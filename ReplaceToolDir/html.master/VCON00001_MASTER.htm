Displaying System Wide Audit Controls

 

The System Wide Audit Controls allow you to display and/or change the audit requirements for the system as a whole.

 

You may supply an Authorisation Code if one is relevant.

 

For VCON Centre users, you may specify the Read From Location and/or the Update Locations to indicate which locations you want the details read from and to which locations you want any updates sent.

 

The system wide audit controls specify:

 

whether or not changes to permissions are to be audited. Auditing should normally be requested

 

whether or not changes to  security classifications are to be audited (HSO only). Auditing should normally be requested

 

whether audit journal messages are to be redirected to the system journal, if the audit journal becomes full, or simply lost. Redirection should be the normal state

 

the Minimum Audit trusts which define the set of trusts for which allowed use will be audited in all VMs;

 

the Default Audit trusts which are those audit trusts given to users or persons when they login if they have not previously been given any explicit audit trusts;

 

the Subject Access Object Auditing which controls system-wide auditing of accesses to catalogued objects;

 

the name of the current audit journal.

 

All these features (except for the name of the audit journal) can be changed using this dialogue box.

 

In addition, you can request a new audit journal using the Switch Audit Journal button (this will be activated immediately). Journals are usually switched automatically by daily house-keeping, but if you need to switch the journal explicitly (e.g. because it has filled up) you may do so. 

 

Minimum Audit Trusts and Default Audit Trusts can be set to 'System High', or 'System Low', or you may define a specific subset of trusts to be audited.

 

Subject Access Object auditing can be set to 'All' or 'None', or you can specify exactly which sorts of access to each type of object that you would like to have audited. You must be careful about the level of Subject Access Object auditing that you apply, because a high level of auditing can decrease system performance. You are very unlikely to need to select 'All'.

 

Press the Update button to get VME to action your changes.

 

Press the Cancel button to exit from the dialogue. If you press Cancel before pressing the Update button no changes will be made to the System Wide Audit Controls.
