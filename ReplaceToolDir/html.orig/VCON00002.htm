Displaying/Changing System Wide Trust Controls

 

The system wide trust controls allow you to specify how the trusts associated with the various catalogued objects (e.g. OMF libraries, Services, SAPs, Tasks, VDUs) are to be used.  

 

You may supply an Authorisation Code if one is relevant.

 

For VCON Centre users, you may specify the Read From Location and/or the Update Locations to indicate which locations you want the details read from and to which locations you want any updates sent.

 

You can specify:

 

whether or not route trusts associated with SAPs and catalogued VDUs are to be used. This facility allows you to control the use of security related commands by users logging in from remote systems. You will need to ensure that the appropriate trusts are put on the DISAPs representing both local and remote systems. See Use of Route Trusts

 

whether or not service trusts associated with MAC, BATCH, and OPER services are to be used. This facility allows you to control which trusted commands can be used from which work management services. See Use of Service Trusts

 

whether or not  trusts associated with Tasks (including System Tasks and Spoolers) are to be used. This facility allows you to control the security related commands that can be called within your system tasks (including those written by your own staff and by third parties). See Use of Task Trusts

 

whether or not trusts associated with OMF libraries that are assigned for execute are to be used. This facility allows you to control which security related commands can be used when certain OMF libraries have been assigned for Execute access. Normally this will not be required. See Use of OMF Library Trusts

 

You can also decide:

 

whether or not loss of trusts is to be block structured. This is normally only necessary if OMF library trusts are being used;

 

whether or not trust violations are to be logged to the VDU screen. This is useful when first introducing the trusts option since users are given immediate notification of which trust they require. (Note that violations are always logged to the audit journal);

 

what public trust overrides are to be used. You can set the public trust overrides to System High or System Low or you can select the specific trusts you want. See Use of Public Trust Overrides.

 

Press the Update button to send your changes to VME.

 

Press the Cancel button to exit from the dialogue. If you press Cancel without pressing the Update button, no changes will be made.
